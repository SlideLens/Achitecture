# Архитектурные диаграммы (C4)

Уровни C1–C3 по [модели C4](https://c4model.com/); классический уровень 4 («код») заменён ERD модели данных — по ней пишутся SQLAlchemy-модели. Плюс диаграмма пайплайна Разбора и sequence ключевого флоу. Термины — по [CONTEXT.md](../CONTEXT.md), решения — по [ADR](../adr/).

> Те же диаграммы есть исходниками в [docs/diagrams/](diagrams/) (формат [d2](https://d2lang.com/)) для сборки «красивого» PDF-отчёта — см. [report/](../report/). Mermaid ниже — рендер-независимый вариант, читается прямо на GitHub.

## C1 — Контекст системы

```mermaid
flowchart TB
    user["👤 Пользователь<br/>консультант / аналитик / сейлз<br/>грузит деки, читает разборы"]

    slidelens["🎯 SlideLens<br/>Агент-ревьюер презентаций:<br/>анализ слайдов, графиков и речи,<br/>автоисправление PPTX"]

    vlm["☁️ Claude API (vision)<br/>анализ слайдов и графиков"]
    email["✉️ Email-сервис (SMTP)<br/>подтверждение почты,<br/>готовность отчёта"]
    obs["📊 Sentry · Langfuse<br/>ошибки, трейсы VLM, стоимость $"]

    user -- "загрузка деки/питча,<br/>чтение отчёта [HTTPS]" --> slidelens
    slidelens -- "промпты + картинки / JSON [HTTPS]" --> vlm
    slidelens -- "письма [SMTP]" --> email
    slidelens -. "ошибки, трейсы, стоимость" .-> obs
```
*Для кого:* стейкхолдер — кто пользуется системой и с чем она общается вовне. Whisper и python-pptx не показаны: это внутренние библиотеки, не внешние сервисы. Observability пунктиром — dev/ops-контур, не рантайм-зависимость (при недоступности Langfuse Разбор продолжается).

## C2 — Контейнеры

Прод: Docker Compose на VPS. Собранный SPA вшит в backend-образ (`./static`) и отдаётся самой FastAPI; Caddy на одном домене терминирует TLS и реверс-проксирует весь трафик на app (без CORS) — см. [DEPLOY.md](DEPLOY.md).

```mermaid
flowchart TB
    user["👤 Пользователь (браузер)"]

    subgraph vps["VPS · Docker Compose"]
        caddy["🔀 Caddy<br/>HTTPS/TLS, один домен:<br/>реверс-прокси всего на app"]
        spa["🖥️ Frontend · React SPA<br/>Vite + TS + Tailwind + TanStack Query<br/>(dist вшит в образ → ./static)"]
        api["⚙️ Backend · FastAPI<br/>JSON API + JWT + лимиты +<br/>приём файлов, отдача отчёта и статики"]
        worker["🔧 Воркер · пайплайн разбора<br/>ARQ (тот же образ, что app)<br/>ingest → анализ → отчёт"]
        db[("🗄️ PostgreSQL<br/>User, Review, Finding,<br/>FileAsset, Event")]
        redis[("⚡ Redis<br/>очередь ARQ")]
        files[("📁 Файлы: диск → S3/R2<br/>оригиналы, PNG, отчёты, fixed.pptx")]
    end

    vlm["☁️ Claude API (vision)"]
    email["✉️ SMTP"]

    user -- "HTTPS" --> caddy
    caddy -- "TLS реверс-прокси" --> api
    api -- "отдаёт SPA из ./static" --> spa
    api -- "SQL" --> db
    api -- "поставить задачу" --> redis
    api -- "сохранить/отдать" --> files
    redis --> worker
    worker -- "vision [HTTPS]" --> vlm
    worker -- "findings, score, status" --> db
    worker -- "артефакты" --> files
    worker -- "отчёт готов" --> email
```
Один origin для API и статики → нет CORS и nginx ([ADR 0004](../adr/0004-stack-fastapi-react.md)). Разбор идёт в воркере, а не в HTTP-запросе ([ADR 0003](../adr/0003-async-review-worker.md)).

*Для кого:* разработчик/DevOps — что реально разворачивается и где живут данные, очередь и файлы.

## C3 — Компоненты бэкенда и пайплайна

Граница жёсткая: `backend/core/` — чистая библиотека, не импортирует `backend/app/` и не ходит в БД; связывает их только воркер ([ADR 0001](../adr/0001-pipeline-pure-library.md)).

```mermaid
flowchart TB
    spa["🖥️ React SPA"]

    subgraph app["backend/app/ · веб-слой (знает про HTTP и БД)"]
        subgraph routers["API-роутеры (/api/v1)"]
            r_auth["auth<br/>register / login / verify"]
            r_rev["reviews<br/>создать, статус, отчёт"]
            r_find["findings · files · events"]
        end
        auth["🔐 fastapi-users<br/>JWT access+refresh, bcrypt"]
        rev_svc["🧩 ReviewService<br/>приём файла, лимиты, enqueue"]
        limits["🚦 LimitService · Storage · EventTracker · EmailService"]
        orm["🗄️ SQLAlchemy 2.0 + Alembic"]
    end

    subgraph worker["backend/worker/ · связующее звено"]
        task["process_review<br/>workdir → Orchestrator → БД/Storage → email"]
    end

    subgraph core["backend/core/ · чистая библиотека (VLM, whisper, pptx)"]
        orch["🎼 PipelineOrchestrator + ReviewContext"]
        llm["☁️-клиент LLMClient<br/>vision, structured output,<br/>retry, стоимость в Langfuse"]
        ingest["📥 DeckIngestor · AudioExtractor · Transcriber"]
        analyzers["🔎 Анализаторы<br/>Slide · Zoom · Deck · Chart · CrossModal"]
        assemble["🧮 Aggregator · DeckScorer · Annotator · PptxFixer · ReportBuilder"]
    end

    db[("PostgreSQL")]
    vlm["Claude API"]

    spa --> routers
    routers --> auth
    r_rev --> rev_svc
    rev_svc --> limits
    rev_svc --> orm
    rev_svc -- "enqueue" --> task
    task --> orch
    orch --> ingest
    orch --> analyzers
    orch --> assemble
    analyzers --> llm
    llm --> vlm
    limits --> orm
    orm --> db
    task -- "findings, score, artifacts" --> orm
```
*Для кого:* разработчик — как декомпозированы веб-слой и пайплайн и где проходит граница «app знает про БД ↔ core чист». Все VLM-вызовы идут через единственный `LLMClient` ([ADR 0002](../adr/0002-vlm-pipeline-hybrid-analyzers.md)).

## Пайплайн Разбора

Десять шагов от загруженной Деки до Отчёта. Падение любого анализатора не валит Разбор — частичный отчёт лучше `failed` ([ADR 0002](../adr/0002-vlm-pipeline-hybrid-analyzers.md)).

```mermaid
flowchart TB
    start(["Review: status = processing"]) --> ingest

    subgraph s1["1. Ингест"]
        ingest["DeckIngestor<br/>PPTX/PDF → PDF (LibreOffice) → PNG (pdf2image)<br/>+ извлечение текстов (python-pptx)"]
        transcribe["Transcriber<br/>аудио → WAV → faster-whisper<br/>→ транскрипт + DeliveryMetrics"]
        excel["Парсинг Excel<br/>(если приложен)"]
    end

    subgraph s2["2. Анализ (VLM = Claude vision)"]
        slide["SlideAnalyzer<br/>иерархия, типографика, читаемость"]
        zoom["ZoomAgent<br/>подозрительные зоны → кроп ×2 → крупно"]
        deck["DeckAnalyzer<br/>консистентность, нарратив, дубли"]
        chart["ChartChecker<br/>обрезанные оси, pie≠100%, сверка с Excel"]
        cross["CrossModalAnalyzer<br/>речь ↔ слайды, рекомендации к деке"]
    end

    subgraph s3["3. Сборка результата"]
        agg["Aggregator + DeckScorer<br/>дедуп, приоритизация, скор 0–100"]
        annot["Annotator<br/>Pillow: рамки-bbox на PNG"]
        fixer["PptxFixer<br/>контраст/выравнивание/шрифт → fixed.pptx"]
        report["ReportBuilder + PdfExporter<br/>HTML-вью-модель + PDF (weasyprint)"]
    end

    ingest --> slide
    ingest -.-> transcribe
    ingest -.-> excel
    slide --> zoom
    zoom --> deck
    slide --> chart
    excel -.-> chart
    transcribe -.-> cross
    deck --> agg
    chart --> agg
    cross -.-> agg
    agg --> annot
    agg --> fixer
    annot --> report
    fixer --> report
    report --> done(["status = done · email пользователю"])
    ingest -->|"ошибка рендера"| fail(["status = failed + fail_reason,<br/>фолбэк: приложить PDF"])
```
*Для кого:* разработчик — порядок шагов и модули пайплайна; пунктир — опциональные ветки (аудио/Excel приложены не всегда).

## Уровень 4 — Модель данных (ERD)

Вместо классического C4-«кода»: по этой схеме пишутся SQLAlchemy-модели. `FindingRow` зеркалит pydantic-`Finding` пайплайна ([ADR 0001](../adr/0001-pipeline-pure-library.md)).

```mermaid
erDiagram
    USER ||--o{ REVIEW : "создаёт"
    REVIEW ||--o{ FINDING : "содержит"
    REVIEW ||--o{ FILE_ASSET : "хранит"
    REVIEW ||--o| REHEARSAL : "фаза 4"
    USER ||--o{ EVENT : "порождает"

    USER {
        uuid id PK
        string email UK
        string password_hash
        string plan "free | paid"
        int free_reviews_left "лимит: 2"
        bool email_verified
        datetime created_at
    }
    REVIEW {
        uuid id PK
        uuid user_id FK
        string status "queued | processing | done | failed"
        int score "0-100, nullable"
        string fail_reason "nullable"
        string deck_filename "имя загруженного файла"
        int n_slides "nullable до успешного ingest"
        json delivery_metrics "nullable, из CrossModalAnalyzer при has_audio"
        bool has_audio
        bool has_data
        datetime created_at
        datetime finished_at "nullable"
    }
    FINDING {
        uuid id PK
        uuid review_id FK
        int slide_num "nullable = уровень деки"
        string category "TYPOGRAPHY|HIERARCHY|READABILITY|CONSISTENCY|CHART|NARRATIVE|SPEECH_MISMATCH|DELIVERY"
        string severity "CRITICAL | MAJOR | MINOR"
        string title "<=80"
        text description
        text fix_suggestion
        json bbox "нормированный 0..1, nullable"
        uuid screenshot_asset_id FK "аннотированный PNG (FILE_ASSET), nullable"
        bool auto_fixable
        bool auto_fixed
        string source "имя анализатора"
        bool user_flag "👎 мусорная находка"
        bool user_like "👍 полезная находка"
    }
    FILE_ASSET {
        uuid id PK
        uuid review_id FK
        string kind "deck_original|slide_png|annotated_png|fixed_pptx|audio|data_xlsx|report_pdf"
        string storage_path
        int size_bytes
        int version "растёт при перегенерации (fixed_pptx на каждый apply_fix)"
        datetime expires_at "автоудаление"
    }
    EVENT {
        uuid id PK
        uuid user_id FK "nullable"
        string name
        json properties
        datetime created_at
    }
    REHEARSAL {
        uuid id PK
        uuid review_id FK
        string audio_path
        json slide_timings "таймкоды переключений"
        json delivery_metrics
        int attempt_num "динамика между прогонами"
        datetime created_at
    }
```

Замечания к модели:
- **Скор и Находки персистятся** — Отчёт (`ReportOut`) собирается из `FINDING` + `FILE_ASSET`, хранится и отдаётся `GET /reviews/{id}/report`.
- **`bbox`** — нормированные координаты `0..1` (не зависят от dpi), по ним рисуется рамка на PNG.
- **`FILE_ASSET.expires_at`** — приватность (US-8): периодическая задача удаляет истёкшие файлы из Storage.
- **`REVIEW.delivery_metrics`** — Подача по приложенной к Разбору Записи питча (MVP, US-2/US-4). Отдельно от `REHEARSAL`, которая остаётся пустой под фазу 4 (запись прямо в браузере с точным `SlideTiming`, динамика между прогонами).
- **`user_flag` (👎) / `user_like` (👍)** — взаимоисключающие голоса; утекают в Langfuse как датасет для итераций промптов ([ADR 0007](../adr/0007-three-layer-observability.md)). Точечный автофикс — `POST /findings/{id}/apply_fix` (набор `auto_fixed`, реген `fixed.pptx` из оригинала).
- **`REHEARSAL`** — пустая под фазу 4 ([ADR 0005](../adr/0005-crossmodal-delivery-analysis.md)); в MVP не заполняется, но заведена, чтобы фича не ломала схему.

*Для кого:* разработчик — по этой ERD пишутся SQLAlchemy-модели напрямую.

## Sequence: путь пользователя (загрузка → отчёт)

```mermaid
sequenceDiagram
    actor U as Пользователь
    participant API as FastAPI
    participant Q as Воркер (пайплайн)
    participant C as Claude API / Whisper
    participant S as БД + файлы
    participant M as Email

    U->>API: Регистрация → подтверждение почты → вход
    U->>API: POST /reviews (дека + аудио? + xlsx?)
    API->>API: проверка лимита (free_reviews_left)
    API->>S: Review(status=queued), сохранить файлы
    API-->>U: 202 — «разбор запущен» (кабинет)
    API->>Q: задача в очередь

    activate Q
    Q->>Q: рендер слайдов в PNG
    Q->>C: Whisper — транскрипт с таймкодами
    loop по каждому слайду + зумы
        Q->>C: VLM-анализ (иерархия, типографика, графики)
        C-->>Q: Findings (JSON)
    end
    Q->>C: межслайдовый анализ + речь ↔ слайды
    Q->>Q: агрегация, скоринг, аннотация, автофиксы → fixed.pptx
    Q->>S: findings, score, artifacts, status=done
    deactivate Q
    Q->>M: «отчёт готов»
    M-->>U: письмо со ссылкой

    U->>API: GET /reviews/{id}/report
    API-->>U: скор, находки по слайдам, блоки «графики / подача / речь↔слайды»
    U->>API: скачать PDF / fixed.pptx
    API-->>U: файлы (проверка владельца)
```
*Для кого:* разработчик — happy path гвоздя продукта; ветка `failed` (ошибка рендера) и ошибки владения (404) показаны в [api/openapi.yaml](../api/openapi.yaml), здесь опущены.
